# 포천아트밸리 모자이크 시스템: 핵심 로직 A to Z

본 문서는 사진 모자이크 전시 시스템의 데이터 흐름과 코어 로직이 동작하는 방식을 A부터 Z까지 완벽히 정리한 명세서입니다. 시각적인 UI 이펙트는 배제하고, 실제 서버와 프론트엔드 간의 데이터 처리 흐름에 집중합니다.

---

## 1. 시스템 아키텍처 및 설정 (Architecture & Config)
시스템은 단일 진실 공급원(Single Source of Truth)인 `config.js`를 바탕으로 모든 로직이 통제됩니다.

- **`config.js`**: 서버 포트(3000), 이미지 리사이징 기준 크기(600px), 백업 보관 일수(7일), 캔버스 크기(1920x1080), 그리고 **모자이크 타일(Grid)의 퍼센트 좌표(`x, y, w, h`)** 및 활성화 상태(`active`)를 정의합니다.
- **`database.json`**: 슬롯의 상태(ID, `filled` 여부, `imagePath`, 업로드 `timestamp`)를 보관하는 인메모리 기반 경량 DB입니다.
- **`server.js`**: 업로드 수신, 이미지 최적화(Sharp), 파일 교체(FIFO), 웹소켓 브로드캐스트, Cloudflare 터널링을 총괄합니다.

---

## 2. 서버 구동 및 일일 자동화 라이프사이클
전시 시스템은 매일 새로운 관람객을 받기 위해 날짜를 추적하고 공간을 자동으로 비웁니다.

1. **날짜 검증 (`checkAndReset`)**: 
   - 서버 기동 시 또는 업로드 발생 시 `database.json`의 `date` 값을 오늘 날짜(YYYY-MM-DD)와 비교합니다.
2. **자동 백업 및 비우기**: 
   - 날짜가 변경되었다면 어제 폴더(`uploads/`)에 쌓인 사진들을 `uploads/backup/{어제날짜}.zip` 파일로 묶어 백업합니다.
   - 백업이 끝난 원본 이미지 파일들은 모두 삭제하여 용량을 확보하고, `database.json`을 빈 상태로 리셋합니다.
3. **만료된 백업 정리 (`cleanOldBackups`)**: 
   - `config.js`에 명시된 보관 기간(기본 7일)이 경과한 zip 백업 파일들은 서버에서 영구 삭제합니다.

---

## 3. 업로드 및 FIFO(선입선출) 교체 로직
관람객이 `upload.html`에서 사진을 전송했을 때 벌어지는 파이프라인입니다.

1. **Multer 수신 및 슬롯 배정**:
   - 이미지를 임시 폴더로 받습니다.
   - `database.json`에서 빈 슬롯(`filled: false`)을 우선 탐색하여 랜덤 배정합니다.
2. **FIFO 덮어쓰기 로직 (꽉 찼을 경우)**:
   - 만약 28칸(또는 활성화된 칸 전체)이 모두 채워졌다면, `timestamp`를 기준으로 정렬하여 **가장 오래된 슬롯(First-In)**을 찾아냅니다.
   - 해당 슬롯에 있던 기존 썸네일 이미지 파일(`fs.rmSync`)을 디스크에서 물리적으로 삭제합니다.
3. **이미지 최적화 (`sharp`)**:
   - 원본(10MB 이상의 고화질 사진)을 `sharp`를 이용해 긴 변 기준 `600px`로 축소하고 JPEG(Quality 85%) 썸네일로 압축합니다.
   - 변환이 끝난 즉시 원본 대용량 파일은 지웁니다.
4. **DB 업데이트 및 실시간 전파**:
   - `database.json`에 새로운 썸네일 경로(`imagePath`)와 현재 시간(`timestamp`), `filled: true`를 기록합니다.
   - 즉시 Socket.io를 통해 `display.html` 화면에 `new_photo` 이벤트와 함께 이미지 경로 및 슬롯 ID를 브로드캐스트합니다.

---

## 4. 디스플레이 화면의 렌더링 및 색상 조화 로직
`display.html`은 전달받은 사진들을 전시 테마(우주)에 어울리게 표출합니다.

1. **DOM 절대 좌표 기반 렌더링**:
   - 무거운 Canvas 렌더링을 피하고 순수 DOM(`<div>`와 `<img>`)을 사용합니다.
   - 서버에서 전달받은 `config.js`의 슬롯별 비율 값(`0.0 ~ 1.0`)에 `100`을 곱하여 CSS `left, top, width, height`를 퍼센트(`%`)로 지정합니다. 이로 인해 브라우저 창 크기가 바뀌어도 레이아웃 비율이 완벽하게 유지됩니다.
2. **이미지 즉각 스왑(Swap)**:
   - 소켓으로 `new_photo` 이벤트가 들어오면 새로고침 없이 해당 슬롯의 `<img>` 태그 `src` 속성만 바꿔치기하여 즉시 사진을 띄웁니다.
3. **오버레이 색상 조화 (Core Blending Logic)**:
   - 사진들의 색감(빨간색, 밝은 낮 등)이 제각각 튀는 것을 방지하기 위한 핵심 처리입니다.
   - 모자이크 컨테이너 위쪽에 원본 우주 배경(`bg.jpg`)을 한 겹 덮고, CSS `mix-blend-mode: overlay` (그리고 `opacity`)를 적용합니다.
   - 이로 인해 뒷면의 사진 픽셀들이 앞면의 우주 색상 픽셀과 섞이게 되어, 사진의 디테일은 보이면서도 전체적인 색감(톤)은 우주 테마로 통일됩니다.

---

## 5. 네트워크 터널링 및 QR 자동 발급 로직
로컬 컴퓨터(전시 PC)에서 구동되는 로컬 호스트를 외부 관람객의 LTE/5G 환경에서 접속 가능하게 만듭니다.

1. **Cloudflare 데몬 실행**: 서버(`server.js`)가 Node 프로세스의 자식 프로세스(`spawn`)로 `cloudflared` 데몬을 백그라운드에서 실행시킵니다.
2. **퍼블릭 URL 캐치**: 자식 프로세스가 내뱉는 로그(stdout) 스트림을 감시하다가, `https://*.trycloudflare.com` 형태의 정규식 패턴이 발견되면 해당 문자열을 임시 퍼블릭 URL로 캡처합니다.
3. **QR 코드 동기화**: 알아낸 URL을 Socket.io의 `tunnel_url` 이벤트로 프론트엔드에 뿌립니다.
4. **자동 그리기**: `guide.html`은 전달받은 URL 문자열(`/upload.html` 경로 포함)을 가지고 즉석에서 Canvas 기반 QR코드를 화면에 그려내어 관람객에게 노출합니다.
