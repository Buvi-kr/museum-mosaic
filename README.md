# 🌌 Photo Mosaic System (V5)

> 현장에서 방문객들의 사진을 모아 거대한 모자이크 아트를 실시간으로 완성하는 전시 솔루션입니다.

## 🚀 1분 만에 실행하기 (초등학생도 할 수 있어요!)

1. **설치하기**: [Node.js](https://nodejs.org/)를 컴퓨터에 설치합니다. (버전 18 이상 권장)
2. **코드 다운로드**: 이 깃허브 페이지에서 초록색 `Code` 버튼을 누르고 `Download ZIP`을 클릭해 압축을 풉니다.
3. **패키지 설치**: 폴더 안에서 주소창에 `cmd`를 입력해 검은 창을 띄운 뒤, `npm install` 이라고 치고 엔터를 누릅니다.
4. **서버 실행**: `start.bat` 파일을 더블클릭합니다! (검은 창이 뜨면서 실행됩니다)
5. **접속하기**:
   - 디스플레이 화면 (큰 모니터용): `http://localhost:3000/display.html`
   - 관리자 설정 화면 (진행요원용): `http://localhost:3000/admin.html`

## 🛠️ 모자이크 슬라이싱 알고리즘 (어떻게 합성될까요?)

이 시스템은 고성능 이미지 처리 라이브러리인 **Sharp**를 사용하여 사진을 실시간으로 자르고 붙입니다!

1. **배경 자동 분할 (Pre-Slicing)**
   - 서버가 켜질 때, `public/backgrounds/` 폴더에 있는 원본 배경 이미지(1920x1080)들을 가져옵니다.
   - 6칸, 12칸, 20칸, 28칸 등 다양한 템플릿 레이아웃의 좌표(x, y)에 맞춰 미리 수십 조각으로 잘라냅니다 (`public/slices/` 폴더에 저장).
2. **사진 즉석 합성 (Real-time Compositing)**
   - 관람객이 스마트폰으로 사진을 찍어 업로드하면, 서버가 즉시 빈 자리를 찾습니다.
   - 해당 자리의 **미리 잘려있는 배경 조각** 위에 관람객의 사진을 `soft-light` 방식으로 자연스럽게 합성합니다. (원본 사진은 `uploads/raw/` 에 보존됩니다!)
3. **디스플레이 즉시 송출 (Socket.io)**
   - 합성된 이미지가 저장되면, 소켓 통신을 통해 화면에 애니메이션 효과와 함께 새로운 사진이 뿅! 하고 튀어나옵니다.

## 🔧 관리자 모드 가이드 (Admin Guide)

관리자 페이지 접속: `http://localhost:3000/admin.html`

1. **그리드 템플릿 변경**: '6칸', '12칸', '20칸', '28칸' 중 하나를 선택하고 [저장]을 누르세요. 저장 즉시 `display.html`이 스스로 새로고침되며 칸 수가 변경됩니다. (기존 조각들은 백그라운드에서 다시 자동 재단됩니다.)
2. **유지할 칸 수 (Keep Count)**: 관람객 사진이 다 찼을 때 완전히 비울지(0), 아니면 몇 개를 살려두고 이어서 진행할지 조절합니다.
3. **새 배경 업로드**: 특별한 날에 맞춘 우주 배경 `.jpg` 또는 `.png`를 업로드하세요. 서버가 알아서 배경을 적용하고, 현재 칸 수에 맞춰 수십 개의 퍼즐 조각으로 쪼개줍니다.

**2. 모자이크 격자 상태 (Grid Layout) 다각화**
- 현장 분위기와 인구 밀집도에 맞춰 큐레이터가 상황을 자유롭게 통제할 수 있도록 **[6칸, 12칸, 20칸, 28칸]**의 총 4가지 전용 하드코딩 템플릿(Grid Template) 라인업 구축 완료 (`src/gridTemplates.js`).
  - **6칸(3x2)**: 관람객 한 명 한 명의 얼굴이 거대하게 노출되는 초고속 회전율 모드
  - **12칸(4x3)**: 적절한 크기와 빠른 회전율을 지닌 캐주얼 모드
  - **20칸(5x4)**: 군중과 개인의 조화가 적절한 황금비율 밀도 모드
  - **28칸(7x4)**: 수많은 사람이 한 화면에 꽉 들어차는 대규모 군중용 파노라마 모드

**3. 무한 사이클(Loop) & 부분 리셋 (Partial Reset) 알고리즘 도입**
- 화면이 100% 꽉 차버린 직후 전부 하얗게 초기화되어 관람객들의 허무함을 유발하던 문제 해결.
- **부분 리셋 로직**: 100% 완성 시 화려한 세리머니 이펙트 후 지정된 대기시간(초)이 지나면, 사전에 세팅한 특정 갯수(예: 절반)만큼만 무작위로 남기고 나머지만 부드럽게 지워내어 다음 관람객이 자연스럽게 이어갈 수 있도록 설계.
- **고해상도 영구 아카이빙**: 화면이 100% 채워지는 영광의 순간마다, 백그라운드 서버 엔진(`sharp`)이 즉시 가동되어 슬롯들의 원본을 병합하여 1장의 1920x1080 "오늘의 엑스포 샷"을 `/history/completions/` 에 영구 저장.

**4. V5 통합 관리자 대시보드 (`/admin`) 및 자동화 엔진 탑재**
- **런타임 Config DB화**: 서버를 껐다 켤 필요 없이 작동 중에 설정을 바꿀 수 있도록 `config.json` 도입.
- **프라이빗 대시보드 (`http://localhost:3000/admin`) 신설**:
  - 관리자가 즉시 **격자 템플릿 변경 (6/12/20/28칸)**, **유지할 칸 수 (Keep Count)**, **세리머니 대기시간**을 조종 가능.
- **오토 슬라이싱 (Auto-slicing) 엔진**: 
  - 과거 터미널(cmd) 창에서 `node slice_bg.js`를 따로 치던 번거로움 완전 제거. 관리자 대시보드에서 새로운 `.jpg/.png` 우주 배경을 업로드하면 서버가 이를 감지해 **즉석에서 1920x1080 해상도 정규화 후 현재 활성화된 템플릿(ex: 20칸)에 맞게 수십 조각으로 자동 재단 및 배포.**
- **동적 실시간 동기화 (Real-Time Sync)**: 관리자가 설정을 저장하거나 배경을 바꾸는 즉시, `Socket.io`의 `config_updated` 이벤트가 발송되어 전시장 화면들이 스스로 새로고침하며 갱신됨.
